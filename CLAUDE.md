# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

ChunkLint is a static analyzer for RAG chunks. It inspects chunk files (JSON/JSONL) and in-memory chunk objects (LangChain `Document`, LlamaIndex node, plain dict) for structural problems before they reach a vector database. No LLM calls, no embeddings, no network — only deterministic heuristics.

Python ≥ 3.10. Distributed as the `chunklint` package with a Typer CLI entry point `chunklint = chunklint.cli:app`.

## Commands

Install for local development (editable, with dev deps):

```bash
python -m pip install -e ".[dev]"
```

Run the CLI:

```bash
chunklint scan examples/bad_chunks.json            # demo fixture
chunklint scan chunks.json --fail-on high --quiet  # CI gate
python -m chunklint.cli scan chunks.json           # fallback if console script not on PATH
```

Tests (pytest is configured via `pyproject.toml` with `testpaths = ["tests"]`):

```bash
python -m pytest                                            # full suite
python -m pytest tests/test_cli.py                          # one file
python -m pytest tests/test_cli.py::test_cli_scan_json_fails_on_high  # one test
```

Lint:

```bash
python -m ruff check .
```

## Architecture

The pipeline is linear: **loader → normalizer → engine → rules → reporter**. Understanding the `Chunk` model and the two rule base classes is enough to add new behavior anywhere in the system.

### Data model (`chunklint/models.py`)

- `Chunk`: pydantic model with `id`, `text`, `source`, `metadata`. Field validators coerce loose input (e.g. `None` text → `""`, non-dict metadata → `{"value": ...}`). Treat this as the single internal representation — every other layer either produces or consumes `Chunk`.
- `Issue`: a single finding (chunk_id, source, rule_id, severity, reason, why_it_matters, fix, snippet).
- `LintReport`: aggregate output with summary counts and the full issue list. `has_high_issues` is the standard SDK gate.
- `LintContext`: thin wrapper that carries the active `ChunkLintConfig` into rule evaluation.

### Normalization (`chunklint/normalizer.py`)

`normalize_chunk` is the central type-dispatch site. It handles `Chunk`, `dict`, LangChain documents (duck-typed via `page_content`), LlamaIndex nodes (duck-typed via `get_content`), and bare objects with a `.text` attribute. New input sources must be added here, not in the engine or the adapters. Source resolution flows through `chunklint/utils/metadata.source_from_metadata` so metadata keys like `file_name`, `path`, `document_id` collapse to a single `source` field.

### Rules (`chunklint/rules/`)

Two base classes in `rules/base.py`:

- `Rule.check(chunk, context) -> list[Issue]` — per-chunk rules.
- `CrossChunkRule.check_all(chunks, context) -> list[Issue]` — rules that need the full ordered list (duplicate detection, adjacent-chunk boundary checks, repeated PDF artifacts).

`rules/__init__.py` exports `DEFAULT_RULES` (per-chunk) and `DEFAULT_CROSS_CHUNK_RULES` (cross-chunk). The engine iterates them in that exact order, so to add a rule: implement the class, add an instance to one of those two lists, and add a default entry to `DEFAULT_RULES` in `chunklint/config.py` so the rule is enabled and has a default severity. Helper `Rule.issue(...)` builds an `Issue` with a snippet auto-generated by `chunklint/utils/text.snippet`.

### Engine (`chunklint/engine.py`)

`lint(chunks, *, config=None, config_path=None)` is the public entry point and is what `chunklint` re-exports from the package root. It normalizes, loads the active config (auto-discovers `chunklint.yml`/`chunklint.yaml` in CWD when no path is given), runs per-chunk then cross-chunk rules respecting `is_rule_enabled`, and returns a `LintReport`. Severity counts come from `chunklint/utils/severity.normalize_severity` — always normalize before comparing.

### Config (`chunklint/config.py`)

`ChunkLintConfig` is pydantic. Rule-level configuration uses `RuleConfig` with `model_config = ConfigDict(extra="allow")`, so rules can read arbitrary extra keys via `config.rule_option(rule_id, key, default)` (e.g. `connector_words`, `ignore_start_words` for `starts_mid_sentence`). User YAML is deep-merged on top of `DEFAULT_RULES`, so partial configs are valid and inherit defaults. `default_config_yaml()` produces the file `chunklint init` writes.

### Adapters (`chunklint/adapters/`)

`langchain.py` and `llamaindex.py` are thin convenience wrappers — they take framework objects and call into `engine.lint` / `loader.export_chunks` after the normalizer handles the actual mapping. The duck-typed normalization in `normalizer.py` is what makes this work without importing the framework packages, which is why the test suite can use fake objects (see `tests/test_adapters.py`).

### Reporter (`chunklint/reporter.py`)

Two distinct outputs share one report:

- Text mode (Rich): groups rules into developer-facing **root causes** defined in `ROOT_CAUSE_SPECS`. Examples and raw rows are hidden by default; `--verbose` reveals examples, `--raw` reveals row-level issues. `print_gate_report` is the variant used when `--fail-on <severity>` is set — it shows the overall summary plus only the root causes for that severity.
- JSON mode: structured payload containing summary, groups, root_causes, recommendations, and raw issues. This is the contract for CI consumers; preserve field names when modifying it.

### CLI (`chunklint/cli.py`)

Typer app with three commands: `scan`, `init`, `rules`. Exit codes are part of the public contract: `0` clean, `1` severity gate failed, `2` invalid input or config (raised as `ValueError` in lower layers), `3` unexpected error. Two mutually exclusive gate flags: `--fail-on` is **exact-severity** (accepts a comma list like `high,medium`); `--fail-on-at-or-above` is **threshold-style** and expands to all severities at or above the given floor. The check is `_has_issues_with_severities` near the bottom of `cli.py`.

## Conventions

- Every module starts with `from __future__ import annotations`. Keep this when adding new files.
- Public surface is `from chunklint import lint, Chunk, Issue, LintReport` (see `chunklint/__init__.py`). Keep this list small and stable.
- Ruff is configured with `line-length = 120` and selects `E, F, I, UP, B, SIM` with `SIM108` and `B008` ignored (see `pyproject.toml`).
- Tests live exclusively under `tests/`. There is one test file per rule family plus separate files for loader, CLI, adapters, and reporter — mirror this layout when adding rules.
